# agent-friendly design

Useful principles for our report output:
- semantic HTML is non-negotiable
- heading hierarchy should be clean
- use native elements instead of div soup
- machine-readable state matters
- content should be understandable even with CSS stripped away

## Why we saved this
Our report generator should produce HTML that is:
- well-structured
- accessible
- easy to inspect and screenshot
- stable for browser-based rendering and future automation

This is less about aesthetics and more about making the HTML foundation solid.
