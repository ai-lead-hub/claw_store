prototype preview / concrete workflow

Switch from generation to whiteboard, publish a shot version, and keep Kitsu as the source of truth.

This prototype focuses on one real flow: generate a shot version, switch into the whiteboard tab, choose the shot number and version number, add a publish comment, and have that publish appear in a vertical shot lineup with horizontal versions. Clicking any shot or version reveals synced comments, so native Kitsu versioning and comments remain the underlying truth.

What this prototype proves

  • The generation workspace and whiteboard can be two tabs in the same product instead of separate tools.
  • The whiteboard can visualize shots vertically and versions horizontally.
  • Publish comments can become visible objects on the board, not buried metadata.
  • Kitsu version IDs and comments can remain canonical while this UI acts like a collaborative creative layer.
